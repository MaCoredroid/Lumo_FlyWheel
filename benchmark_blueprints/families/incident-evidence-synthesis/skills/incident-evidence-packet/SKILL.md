# Incident Evidence Packet

Use this skill when a task requires a concise postmortem-style synthesis from frozen incident evidence.

## Workflow
1. Separate trigger, symptom, guardrail failure, and follow-up.
2. Cross-check logs against tickets and timeline notes.
3. Mark unresolved ambiguity explicitly instead of guessing.
4. Keep the follow-up recommendation bounded to the proven facts.

## Anti-Patterns
- Treating the first ticket as the ground truth.
- Collapsing symptom and cause.
- Hiding uncertainty behind generic language.

## Done Signal
- The packet names the trigger, guardrail failure, and bounded follow-up with explicit evidence.
