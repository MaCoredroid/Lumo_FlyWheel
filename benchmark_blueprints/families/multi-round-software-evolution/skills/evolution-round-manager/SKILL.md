# Evolution Round Manager

Use this skill when a task requires choosing the next round of long-horizon software evolution.

## Workflow
1. Read prior rounds and objective history before choosing a focus.
2. Prefer moves that unlock future progress over flashy local cleanup.
3. Define explicit do-not-touch boundaries.
4. Use a partial-progress metric that can credit meaningful movement.

## Anti-Patterns
- Planning a round with no boundary.
- Chasing the most visible issue instead of the true blocker.
- Using a vague success metric.

## Done Signal
- The next-round plan is focused, bounded, and aligned to measurable progress.
