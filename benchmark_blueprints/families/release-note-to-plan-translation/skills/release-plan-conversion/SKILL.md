# Release Plan Conversion

Use this skill when a task requires turning release notes and repo evidence into an implementation plan.

## Workflow
1. Extract the intended outcomes from the release notes.
2. Build the dependency order from the repo inventory.
3. Keep the first milestone small and meaningful.
4. Surface user-visible risk explicitly.

## Anti-Patterns
- Translating bullets into an unordered checklist.
- Choosing a first milestone that is too broad.
- Ignoring implicit risk in repo state.

## Done Signal
- The plan is ordered, bounded, and grounded in both release notes and repo evidence.
