# Root Cause Archaeology

Use this skill when a task requires reconstructing causality from frozen issue, PR, and commit history.

## Workflow
1. Build a timeline before drafting a conclusion.
2. Separate originating changes from downstream discussion.
3. Reject at least two plausible but wrong suspects with evidence.
4. Keep the recommendation bounded to the proven root cause.

## Anti-Patterns
- Following the loudest discussion thread.
- Confusing first report with first cause.
- Recommending a broad refactor without proving the trigger.

## Done Signal
- The memo identifies the earliest cause, rejects downstream suspects, and recommends a bounded next step.
