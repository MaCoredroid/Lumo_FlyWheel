# Backlog Dependency Plan

Use this skill when a task requires ordering a backlog under dependency and team-capacity constraints.

## Workflow
1. Identify true blockers and prerequisites first.
2. Respect scarce specialist capacity.
3. Sequence observability and risk-reduction work before high-risk rollout items.
4. Optimize for objective progress, not only raw item count.

## Anti-Patterns
- Scheduling blocked work first.
- Ignoring specialist capacity.
- Front-loading risky rollout work without guardrails.

## Done Signal
- The schedule is dependency-aware, capacity-aware, and strategically ordered.
