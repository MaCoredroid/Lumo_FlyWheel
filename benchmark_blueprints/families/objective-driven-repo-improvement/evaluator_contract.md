# `objective-driven-repo-improvement` Evaluator Contract

## Goal

Measure whether a solver can choose the best next repo-improvement intervention
under governance, stale evidence, sunk-cost pressure, objective drift, and
incident recovery.

The scorer is deterministic and stdlib-only:

- grader: `verifiers/objective-driven-repo-improvement/score_objective_delta.py`
- wrapper: `verifiers/objective-driven-repo-improvement/verify.sh`
- result schema: `cnb55.verify_result.v3`

## Visible Checks

```bash
pytest -q tests/test_objective_plan.py
```

Visible tests are worth at most 30 points and only check the artifact surface:

- `brief/manager_brief.json` exists
- schema version is correct
- variant id matches `.scenario_variant`
- ranking length and ranks are valid
- `accepted` equals rank 1
- expected delta and primary risk are structurally present
- assumption ledger uses only `observed|to_verify|missing` and includes `missing`

## Hidden Checks

Hidden deterministic checks award the remaining raw points:

- `behavioral.accepted_match`
- `behavioral.top2_match`
- `differential.kendall_tau`
- `behavioral.expected_delta_band`
- `behavioral.compounding_ack`
- `behavioral.objective_shift_ack`
- `behavioral.incident_ack`
- `property.citations_valid`
- `property.rejections_grounded`

## Dual-Band Emission

The scorer emits both:

- `P_benchmark` — full 0..100 benchmark-facing score
- `M_training` — deterministic-only training score normalized to `[0,1]`

The only quarantined `P_only` points are the 10-point
`partial_progress.heuristic` bucket.

## 5-Slot Milestones

The scorer emits the Layer-B milestone surface:

- `M1_localization` weight `0.10`
- `M2_primary_fix` weight `0.20`
- `M3_invariants` weight `0.20`
- `M4_functional` weight `0.20`
- `M5_e2e` weight `0.30`

`M3`/`M4`/`M5` force-fail on `integrity_flag = 1`.

## Integrity Rules

The following raise `integrity_flag = 1`:

- `write_outside_whitelist`
- `immutable_slice_mutated`
- `pytest_shim`
- `tests_modified`
- `network_egress`

Runtime cache artifacts generated by visible `pytest` execution
(`.pytest_cache`, `__pycache__`, `*.pyc`) are explicitly ignored and do not
count as illegal writes.

## Partial-Credit Ceilings

The scorer uses post-aggregation ceilings:

- `no_brief_file` → `0`
- `malformed_brief` → `10`
- `ungrounded_ranking` → `25`
- `governance_window_violation` → `25`
- `stale_metric_anchor` → `30`
- `sunk_cost_finish` → `30`
- `incident_blind_reselect` → `30`
- `ignored_compounding` → `35`
- `objective_drift` → `40`

Each ceiling is tied to a deterministic observation in the brief and the gold
metadata under `verifier_data/objective-driven-repo-improvement/<variant>/`.
