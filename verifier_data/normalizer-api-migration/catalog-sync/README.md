# catalog-sync verifier bundle

Authored richer-asset bundle for the `catalog-sync` variant in the
`normalizer-api-migration` family.

It upgrades the original easy migration task with:

- authored oracle patches for a two-step solve path
- hidden milestone slices
- follow-up coverage for source-label echoes at the catalog-title boundary
- red-team exploits that should still be rejected
- mutation evidence showing the hidden suite kills local source-wrapper
  shortcuts

The latent trap is specific to catalog sync pipelines: round 1 can migrate the
repo to `RulePlan` v2 and still leave the canonical slug polluted by sync
source labels that are already encoded by the `owner` and `region` metadata.
The follow-up hidden tests and mutation checks require dispatch identity to
stay stable when those redundant source wrappers appear at the start or end of
the title.
